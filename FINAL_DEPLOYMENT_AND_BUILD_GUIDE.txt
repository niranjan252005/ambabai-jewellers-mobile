PART 2: BUILD MOBILE APP IN VS CODE
========================================

PREREQUISITES:
- VS Code installed
- Flutter extension for VS Code
- Android Studio (for Android SDK)
- Flutter SDK installed

METHOD 1: VS CODE FLUTTER EXTENSION (RECOMMENDED)
========================================

STEP 1: Install Flutter Extension
- Open VS Code
- Go to Extensions (Ctrl+Shift+X)
- Search "Flutter"
- Install "Flutter" extension by Dart Code
- Also install "Dart" extension

STEP 2: Open Mobile Project
- File > Open Folder
- Navigate to: D:\jewellery\mobile
- Click "Select Folder"

STEP 3: Setup Flutter SDK Path
- Press Ctrl+Shift+P
- Type "Flutter: Change SDK"
- Select your Flutter SDK path (e.g., C:\flutter)

STEP 4: Get Dependencies
- Open terminal in VS Code (Ctrl+`)
- Run: flutter pub get

STEP 5: Build APK using VS Code
- Press Ctrl+Shift+P
- Type "Flutter: Build APK"
- Select "Flutter: Build APK"
- Wait for build to complete (5-10 minutes)
- APK will be at: build/app/outputs/flutter-apk/app-release.apk

ALTERNATIVE: Use Terminal in VS Code
- Open terminal (Ctrl+`)
- Run: flutter build apk --release

METHOD 2: VS CODE INTEGRATED TERMINAL
========================================

STEP 1: Open Project in VS Code
- File > Open Folder
- Select: D:\jewellery\mobile

STEP 2: Open Integrated Terminal

STEP 3: Verify Flutter Installation

flutter doctor

STEP 4: Install Dependencies
flutter pub get

STEP 5: Generate App Icons
flutter pub run flutter_launcher_icons:main

STEP 6: Build Release APK

========================================
PART 7: TROUBLESHOOTING
========================================

IF FLUTTER BUILD FAILS:
1. Run: flutter clean
2. Run: flutter pub get
3. Run: flutter doctor
4. Fix any issues shown
5. Try build again

IF VS CODE DOESN'T DETECT FLUTTER:
1. Install Flutter extension
2. Set Flutter SDK path
3. Restart VS Code
4. Run: flutter doctor

IF APK WON'T INSTALL:
1. Enable "Unknown sources" in Android settings
2. Check Android version (requires 5.0+)
3. Free up storage space
4. Try different transfer method

IF GITHUB PUSH FAILS:
1. Run: git pull origin main
2. Resolve any conflicts
3. Run: git push origin main
4. Or use force push: git push origin main --force
